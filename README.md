# Pubblicare immagini Docker

_Impara a creare e pubblicare immagini Docker con GitHub Actions: dai comandi di base all'automazione con azioni Docker specializzate._

</header>

## Benvenuto

- **A chi è rivolto**: studenti, sviluppatori e ingegneri DevOps che vogliono automatizzare la creazione e la pubblicazione di immagini Docker.
- **Cosa imparerai**: come creare e pubblicare immagini Docker con GitHub Actions, usando versionamento automatico e un flusso di rilascio completo.
- **Cosa costruirai**: un workflow di GitHub Actions che crea e pubblica automaticamente immagini Docker su GitHub Container Registry.
- **Prerequisiti**:
  - Familiarità con i workflow di GitHub Actions al livello dell'esercizio [Hello GitHub Actions](https://github.com/skills/hello-github-actions)
  - Familiarità base con la containerizzazione e [Docker](https://docs.docker.com/get-started/docker-overview/)

- **Quanto dura**: Questo esercizio richiede meno di 1 ora per essere completato.

In questo esercizio:

1. Preparerai un ambiente di sviluppo e creerai un workflow Docker di base per pubblicare su GitHub Container Registry.
1. Migliorerai il workflow con azioni Docker specializzate.
1. Implementerai il tagging dinamico per il versionamento automatico basato sul repository Git.
1. Farai pratica con un ciclo di sviluppo completo: funzionalità, pull request, release e versionamento dei container.

### Come iniziare questo esercizio

1.  Fai clic con il tasto destro su **[Step 0](../../actions/workflows/0-start-exercise.yml)** e aprilo in una nuova scheda.
2.  Fai clic su **Run workflow**.
3.  Seleziona il branch `main` e fai clic sul pulsante verde **Run workflow**.
4.  Attendi circa 20 secondi, poi aggiorna questa pagina.
